# 🔁 Kong Exit Transformer Plugin Demo

This project demonstrates the use of Kong's **exit-transformer plugin** to **convert upstream HTTP status code `403` to `401`** in specific scenarios.

## 🛠️ Service and Route Setup

### ✅ Create the Service

```bash
curl -s -X POST http://localhost:8001/plugin-demos/services \
  -H 'Kong-Admin-Token:<admin-token>' \
  -d "name=exit_transformer_demo_svc" \
  -d "url=https://httpbin.konghq.com/status"
```

**Response:**

```json
{
  "name": "exit_transformer_demo_svc",
  "host": "httpbin.konghq.com",
  "path": "/status",
  ...
}
```

### ✅ Add a Route

```bash
curl -i -X POST http://localhost:8001/plugin-demos/services/exit_transformer_demo_svc/routes \
  -H 'Kong-Admin-Token:<admin-token>' \
  --data "name=exit_transfomer_demo_rt" \
  --data "paths[]=/http-exit-code"
```

**Response:** `201 Created` — Route successfully created.

## ⚙️ Add Exit Transformer Plugin

### Plugin Configuration

```bash
curl -i -X POST http://localhost:8001/plugin-demos/services/exit_transformer_demo_svc/plugins \
  -H 'Kong-Admin-Token:<admin-token>' \
  -H "Content-Type: application/json" \
  -d '{
    "name": "exit-transformer",
    "config": {
      "functions": [
        "return function(status, body, headers) if status == 403 then return 401, body, headers end return status, body, headers end"
      ]
    }
  }'
```

**Response:** Plugin added successfully.

---

## 🔁 Retest After Plugin Application

```bash
curl -i http://localhost:8000/http-exit-code/403
```

**Response:** Still returns `403`. ❌ Transformation not applied.

### 🔍 Why?

The plugin does not catch `403` errors *directly from the upstream*. Exit Transformer plugin cannot catch errors originating from upstream services. The Exit Transformer plugin only transforms responses that are generated by Kong itself (i.e., when kong.response.exit() or kong.response.error() is called internally by Kong or its plugins). A 403 returned directly from your upstream service would pass through to the client unchanged.

If you need to modify a status code coming from an upstream service (e.g., transform a 403 response), you would need to use a different approach, such as the post-function serverless plugin in the header_filter phase.

## 🔐 Add OIDC Plugin

```bash
curl -i -X POST http://localhost:8001/plugin-demos/services/exit_transformer_demo_svc/plugins \
  -H 'Kong-Admin-Token:<admin-token>' \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=openid-connect" \
  -d "config.client_id=<client-id>" \
  -d "config.client_secret=<client-secret>" \
  -d "config.issuer=<issuer>" \
  -d "config.auth_methods[]=client_credentials" \
  -d "config.auth_methods[]=bearer" \
  -d "config.auth_methods[]=introspection"
```

### Test OIDC Authenticated Requests

All calls return original upstream responses (`200`, `401`, `403`).

## 🚫 Add ACL Plugin to Deny OIDC

```bash
curl -i -X POST http://localhost:8001/plugin-demos/services/exit_transformer_demo_svc/plugins \
  -H 'Kong-Admin-Token:<admin-token>' \
  --data "name=acl" \
  --data "config.deny=openid"
```

## ✅ Plugin Effect Verification

### Request With Valid OIDC Token (Expecting 403 from ACL)

```bash
curl -i http://localhost:8000/http-exit-code/403 \
  -H "Authorization: Bearer <valid-token>"
```

**Transformed Response:**

```http
HTTP/1.1 401 Unauthorized
{"message":"You cannot consume this service"}
```

✅ **Exit transformer plugin successfully transformed the 403 response (from ACL) to 401.**

---

## ❌ Disable Exit Transformer and Retest

Now disable the plugin or delete it. Then test again:

### Expectation: 403

```bash
curl -i http://localhost:8000/http-exit-code/403 \
  -H "Authorization: Bearer <valid-token>"
```

**Actual Response:**

```http
HTTP/1.1 403 Forbidden
{"message":"You cannot consume this service"}
```

✅ Confirms that **transformation was being done by exit-transformer** plugin.

---

## 📌 Conclusion

* The **exit-transformer plugin works**, but **only transforms Kong-generated errors (like from ACL plugin)**.
* It **does not transform errors directly returned from the upstream service** (like httpbin).
